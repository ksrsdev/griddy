Your Assignment: The Queue Library

Your goal is to create a small "API" for your FIFO list. Instead of juggling head and tail pointers directly in main, you’ll put them inside a "Manager" struct.
The Requirements:

    The Structs: Keep Node, but add a Queue struct:
    C

    typedef struct {
        Node *head;
        Node *tail;
    } Queue;

    The Functions: Implement these three functions:

        void enqueue(Queue *q, const char *data): Handles the malloc, strdup, and the logic of attaching the node to the tail.

        void printAndClear(Queue *q): Prints the strings and frees all memory, ensuring head and tail end up as NULL.

        void initQueue(Queue *q): A simple function to set the initial head and tail to NULL.

What your main should look like:
C

int main(void) {
    Queue myQueue;
    initQueue(&myQueue);

    // ... loop to get input ...
    // Inside the loop: enqueue(&myQueue, buffer);

    printAndClear(&myQueue);

    return 0;
}

Why this is harder than it looks:

You'll have to deal with Pass-by-Reference. When you pass &myQueue to a function, that function receives a pointer to your struct. You'll be using the "arrow operator" (->) a lot (e.g., q->head = temp).
One Hint for the Professional Touch:

In your enqueue function, if the list is empty, both q->head and q->tail need to point to the new node. If it's not empty, only q->tail moves. This is where your previous logic will live!
